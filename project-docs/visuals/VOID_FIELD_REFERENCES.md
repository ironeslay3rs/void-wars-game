\# VOID FIELD REFERENCES



\## Purpose

Defines what a Void Field should feel like.



\## Principles

\- Feels like a place, not a menu

\- Open movement space

\- Landmarks at edges

\- Combat readable at all times



\## Entity Rules

\- Player clearly visible

\- Mobs clearly visible

\- Target instantly recognizable



\## Behavior

\- Movement is direct and responsive

\- Combat is readable

\- Feedback is immediate



\## DO NOT

\- build UI-heavy overlays

\- block movement with UI layers

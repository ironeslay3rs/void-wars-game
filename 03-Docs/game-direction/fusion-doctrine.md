# Void Wars: Oblivion — Fusion Doctrine

## Purpose
Lock one implementation doctrine for the official launch merge so borrowed systems improve readability and retention **without replacing** Void Wars identity.

## Locked Source-of-Truth Order

1. **Book / Void Wars canon = identity law (non-negotiable)**
2. **Overmortal patterns = progression spine**
3. **AFK Journey patterns = presentation and field readability layer**

If any borrowed pattern conflicts with canon truth, canon wins.

---

## Identity Law (Must Stay Void Wars)

These anchors are mandatory for all launch-facing features:

- Void = 3D prison
- Fusion is the long-arc truth (body + mind + soul)
- Macro schools remain **Bio / Mecha / Pure** in player-facing naming
- Puppy start and fragile early loop
- Black Market as neutral survivor citadel
- Consequence systems: danger, pressure, corruption risk, overload/fracture pressure
- Player-driven economy and role value (fighters, gatherers, crafters, brokers, hybrids)

### Naming guardrail
Some planning language may still mention legacy "Spirit." Launch-facing implementation must use **Pure** to stay canon-safe with current registry and faction law.

---

## Borrowing Doctrine

### Overmortal-style systems (Allowed scope)
Use for progression structure only:

- chapter-based advancement framing
- daily return cadence
- breakthrough pacing
- rank and long-arc ascent clarity
- repeatable personal growth rhythm

Do **not** borrow world identity, faction identity, or replacement lore.

### AFK Journey-style systems (Allowed scope)
Use for readability/presentation only:

- clearer map and zone readability
- cleaner home -> field -> reward -> upgrade -> repeat flow
- stronger AFK payout readability
- encounter/payout panel clarity
- smoother route and next-action comprehension

Do **not** soften Black Market tone into generic cozy fantasy.

---

## Explicit "Do Not" Rules

- Do not merge identities into a clone feel.
- Do not flatten Bio/Mecha/Pure into color swaps.
- Do not jump to giant war-map complexity before core loop quality.
- Do not hide the world behind menu bloat.
- Do not let borrowed systems rewrite canon labels, factions, or cosmology.

---

## Official Launch Fusion Formula

### Keep from Void Wars
- world, lore, faction law, terminology
- school identity and school-specific problem solving
- risk/economy/profession depth

- long convergence fantasy

### Borrow from Overmortal
- progression spine

### Borrow from AFK Journey
- readability layer

## Delivery Phases (Execution Order)

### Phase A — Lock merge doctrine (this file)
Goal: one rule-set all teams follow before more implementation.

### Phase B — Rebuild first 30-minute loop
Target player sequence:

1. arrive as Puppy
2. first rumor / bad deal
3. first deployment
4. first recovery
5. first school-shaped choice
6. first Black Market broker interaction
7. first mark eligibility gate

### Phase C — Refactor shell only after loop is fun
Architecture enforcement:

- `app/` thin route wrappers only
- screen composition in `components/`
- reusable UI in shared components
- rules/values in `features/`
- asset paths via `lib/assets.ts`

### Phase D — AFK clarity layer
Add after loop quality is validated:

- idle reward clarity panels
- expedition timers
- passive upkeep/readouts
- resumable state cues
- clearer world travel prompts

### Phase E — War layer expansion
Add later:

- contested zones
- faction pressure escalators
- guild contribution depth
- arena prestige pressure
- Rune Crafter / Rune Knight economy depth

## Merge Acceptance Checks

Every fusion task should pass all checks:

1. Does this preserve Void Wars canon language and tone?
2. Is this borrowing structure/readability rather than identity?
3. Does this improve "what changed / what next" clarity?
4. Does this keep survival pressure visible?
5. Does this avoid genericization?

If any check fails, revise before merge.

## One-line Team Direction

Use Overmortal to structure growth, AFK Journey to structure readability, and the book canon to define truth.

# Void Wars Visual Direction Rule

## North-star reference
Use the menu concept image shared in the task conversation as the **primary visual benchmark** for the home screen and future HUD-heavy screens.

## Rule
When designing or revising major UI surfaces, prefer decisions that move the experience **closer to the reference image** and avoid choices that make the interface feel generic, flat, or modern-SaaS.

## Non-negotiable qualities
- **Dark sci-fi fantasy tone** with a moody, high-contrast city/void backdrop.
- **Layered HUD framing** with beveled panels, angular borders, inset shadows, and metallic trims.
- **Faction-first identity** where Bio, Mecha, and Spirit are immediately readable through color, iconography, and portrait treatment.
- **Cinematic center stage** that gives the main scene or hero illustration the most visual weight.
- **Diegetic game-menu feeling**: every card, button, and meter should look like it belongs inside a game interface, not a dashboard.
- **Glowing resource accents** using controlled neon reds, cyans, greens, and violets on top of a dark base.
- **Strong typography hierarchy** with bold uppercase labels for navigation, panel headings, and key actions.

## UI decision filters
Before shipping a visible UI change, check that it:
1. Increases atmosphere, depth, or faction flavor.
2. Preserves a readable silhouette with distinct left rail, center focus, and right rail zones when relevant.
3. Uses chrome, texture, glow, or framing with restraint so the screen feels premium rather than noisy.
4. Supports the game's combat-fantasy identity better than a minimal or utility-first alternative.

## Anti-patterns
Avoid:
- flat white cards on clean neutral backgrounds;
- rounded consumer-app styling that weakens the combat HUD aesthetic;
- generic placeholder illustrations when faction or world art can be used;
- low-contrast text or effects that reduce play readability;
- color choices that blur the distinction between Bio, Mecha, Spirit, and Void resources.

## Practical application
- Treat the current home/menu experience as the first surface that should converge toward this rule.
- Reuse this direction for status, missions, bazaar, arena, and faction screens so the whole game feels like one unified world.
- If a proposed UI change improves usability but conflicts with the visual direction, solve for both instead of dropping the theme.
